# fl_comp_automata
Automata/machines I made for my Formal Language and Computability class using the Jflap software

DFAs_NFAs folder holds DFAs and NFAs (Deterministic and Nondeterministic Finite Automata) we made in class (the most basic machines we'll ever make)

PDAs folder holds any Pushdown Automata I made (this machine adds a storage mechanism to the automata, a stack that involves pushing and popping values)

TuringMachines folder holds Turing Machines I've worked on, including from exercises and projects (new machine that provides read and write and direction to go after read and write - no stack though)


You will need the Jflap software to run these machines. When first opening Jflap, you just need to click 'File' in the top left corner, 'Open', then find these files (wherever you put them in, having a Github folder to easily navigate to worked best for me) and click 'Open' on each/any selected .jff file to open said automata. 

P.S. the read_this files in each folder gives context to any important machines (the specific problem for them/the language you're solving for with the machine)
